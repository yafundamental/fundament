# Основные сведения
1. так назыаемые "легковесные потоки"
2. являются основным инструментом в Golang для осуществления конкурентного исполнения
3. исполнение goroutine'ами контролируется планировщиком Golang
4. у каждой goroutine'ы есть свой stack с изначальным размером в 2 кБ
5. стек gorotuine'ы может увеличиваться и уменьшаться по мере необходимости
6. по сравнению с thread операционной системы, переключение между двумя goroutine'ами происходит значительно быстрее
7. программист может контролировать исполнение goroutine'ин при помощи контекстов (context)
